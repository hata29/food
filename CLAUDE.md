# Manami's Recipe Kit - 運用ルール

このリポジトリはManamiの個人**食生活管理**用（自炊レシピ + 外食リスト + 週次運用）。

スコープ：
- **recipes/**: 自炊レシピ
- **restaurants/**: 外食リスト（行きつけ・お気に入りの店）
- **weekly/**: 週次の献立＋買い物リスト＋振り返り（1週=1ファイル）

完璧を目指さず、運用しながら育てる。最初の入力は雑でいい、3行でもOK。

## 読み書きの役割分担

- **PCのClaude Code**（`cd ~/projects/life/food && claude`）：レシピ追加・更新、献立組成、外食リスト追記、コミット&push 全般
- **スマホClaudeアプリ / Web版claude.ai**：レシピ閲覧、「何作れる？」相談、献立・買い物リスト確認（**読み取り専用**）

> ※スマホ・Web版の書き込みはClaude側のGitHub Connectorの制約で現状不可（本来はスマホでも書ければ理想）。Connectorが書き込み対応したら運用見直す。

## 基本方針

- **一人暮らし・平日忙しい**が前提
- 自炊の目安：平日夜2〜3食 + 平日昼2〜3食（在宅日のみ）
- レシピは1人分ベースで記述（必要に応じて2食分作って冷凍も検討）
- めんどくさがり度を考慮し、工程数・洗い物が少ない方向に最適化
- **食材の使い切りを最優先**（食材ロスは自炊継続の最大の挫折要因）
- Manamiが作って美味しかった実体験のフィードバックを蓄積していく

## 好み・既知の情報

### 食の好み・嫌い
- 辛いの大好き🔥（クミン・カイエン増量OK）
- 味付けはガツン系・濃いめ・りゅうじ系が好き。りゅうじレシピをよく参考にしている
- カルボナーラ的な重い洋食は気分次第（合わない時もある）
- **生トマトは×**（加熱はOK）。それ以外は基本何でもいける

### 偏愛の食材・調味料
- 大葉（多用）
- 桃屋の刻みニンニク（時短の味方）
- 豚バラ薄切り
- しらす
- ベーコン
- ごま油・オリーブオイル
- めんつゆ
- オイスターソース（つけ汁の隠し味として発見）
- 七味（ペペロンに合う）

### お酒（晩酌するタイプ）
- オレンジワインにハマり中🍷
- 日本酒、ハイボール、ビールも飲む

### 生活パターン
- 手持ち調理器具：電子レンジ、炊飯器（タージン型）、フライパン、鍋
- 在宅勤務週4。出社日の昼は社食前提
- 平日夜：帰宅21〜22時もザラ、自炊15分以内が理想
- 平日昼：在宅日のみ自炊、10分以内、作り置き＋ご飯もOK
- 朝食はスコープ外
- 木金夜は最初から外食・コンビニOK枠

## ディレクトリ構造

```
recipes/         # 完成した自分のレシピ（Markdown）
restaurants/     # 外食リスト（restaurants/{ジャンル}/{店名}.md。ジャンルディレクトリは使う時に作る）
weekly/          # 週ごとの献立＋仕込み＋買い物リスト＋繰越＋振り返り（YYYY-Www.md、1ファイル統合）
inbox/           # 「あとで整形して」用のURL・メモ置き場
references/      # 参考にした他人のレシピメモ（出典明記）
```

## レシピのフォーマット

`recipes/` に保存するファイルは以下のフォーマットに従う：

```markdown
# レシピ名

## メタ情報
- カテゴリ: 平日 / 休日 / おつまみ / 作り置き
- 所要時間: ○分
- 一人分量: ○人前
- タグ: #カレー #トマト #ひき肉 など

## 材料（1人分）
- 材料名: 分量

## 手順
1. ○○する
2. ○○する

## メモ・自分の気づき
- 「○○すると神」など実際に作った時の発見
- 失敗パターン
- 味変・アレンジのアイデア

## 出典
- オリジナル / 参考URL / アレンジ元
```

## 外食リストのフォーマット

`restaurants/{ジャンル}/{店名}.md` で保存：

```markdown
# 店名

## 基本情報
- ジャンル：
- 立地・最寄り駅：
- 営業時間：
- 予算：

## お気に入りメニュー
- ⭐ メニュー名

## シーン（複数可、追記式）
- 一人飲み / ご褒美 / 仕事後 / デート など

## 学び・メモ
- 訪問して気づいたこと

## 訪問履歴
- YYYY-MM-DD：何を食べた、どうだった
```

最初の入力は雑でOK。3行でも記録する。シーンは累積式で増やしていく。

## 攻略書フォーマット（晩酌セット型・複数品同時並行）

複数品を効率的に並行作業するレシピ用の特殊フォーマット。`recipes/` に他レシピと同じ位置で保存。

```markdown
# レシピ名（〇品同時並行攻略版）

## メタ情報・全体メニュー
## 買い物リスト・調味料チェック
## 全食材・分量まとめ（保存版）
## 作業順序＆並行作業（時系列で）
## タイムライン早見表
## お酒のペアリングガイド（あれば）
## 翌日リメイク
## メモ・気づき
```

通常のレシピフォーマットでは表現しきれない並行作業の段取りを「作戦書」として残す用途。

## Claudeの動き方ルール

### レシピ追加時
1. 既存の `recipes/` を確認して重複・類似レシピがないかチェック
2. URLが渡された場合は `web_fetch` で取得し、上記フォーマットに整形
3. ファイル名は `料理名.md`（日本語OK、スペースはハイフン）
4. 追加後はGitHubにコミット&プッシュ

### レシピ更新時（Manamiが「○○のレシピに追記して」と言った時）
1. 該当レシピを読み込み
2. 「メモ・自分の気づき」セクションに追記（既存内容は消さない）
3. コミットメッセージは `update: ○○ - 追記内容の要約`

### 外食リスト追加・更新時
1. `restaurants/{ジャンル}/{店名}.md` で保存。ジャンルディレクトリは初出時に作る
2. 既存ファイルへの追記：訪問履歴・シーン・学びを累積式で追加（既存内容は消さない）
3. シーンは複数可。後から用途別の絞り込みに使う
4. 完璧を目指さない、3行でも記録する

### ハイブリッド提案時（「今夜どうしよ？」「今日何食べる？」）
1. `recipes/` と `restaurants/` の**両方を参照**して候補を出す
2. 体調・気分・時間・冷蔵庫の中身（必要なら聞く）を考慮
3. **自炊と外食を両方提示する**（一方に絞らない）
4. 自炊側は所要時間・買い物要否で絞る、外食側はシーン適合で絞る

### 献立作成時（「今週の献立組んで」と言われた時）

#### 最初に確認すること
- **食数**：平日夜・平日昼・休日それぞれ何食分必要か都度聞く（固定しない）
- **稼働状況**：忙しめ週か余裕週かのヒントがあれば反映

#### 組み方の優先順位（高い順）
1. **食材使い切り最優先**：半端食材（大葉1束・長ねぎ1本・トマト缶等）は2レシピ以上で消費する構成にする
2. **候補レシピ不足時の補完**：手持ち`recipes/`で組めない場合、新レシピを1〜2品提案する（フォーマットに整形してrecipes/に追加）
3. **連続回避はゆるく**：同じタンパク源が3日連続するのは避ける程度。それ以上はこだわらない
4. **日曜の仕込み枠**（30分以内）も提案する。来週の時短に効くもの優先
5. **季節食材を意識**：下記の月別ヒント参照（目安、厳密ではない）

#### 平日昼の扱い
- 対象：在宅勤務日のみ（週2〜3食目安）
- 制約：**調理時間10分以内**
- 形式：軽食（そうめん・蕎麦・冷凍うどん等）または作り置き＋ご飯
- 朝食はスコープ外

#### 季節食材ヒント（目安）
| 月 | 旬の食材 |
|---|---|
| 1〜2月 | 大根・白菜・ほうれん草・ぶり・牡蠣 |
| 3〜4月 | 新玉ねぎ・新じゃが・春キャベツ・アスパラ・たけのこ |
| 5月 | 新玉・アスパラ・空豆・ふき・初鰹 |
| 6〜7月 | トマト・ピーマン・なす・きゅうり・とうもろこし |
| 8月 | なす・ゴーヤ・オクラ・トマト・枝豆 |
| 9〜10月 | さつまいも・きのこ・秋刀魚・鮭・栗 |
| 11〜12月 | 白菜・春菊・れんこん・ぶり・長ねぎ |

#### 出力先（統合：1週=1ファイル）
`weekly/YYYY-Www.md` に **献立＋仕込み＋買い物リスト＋翌週繰越＋メモ振り返り** を1ファイルでまとめる。ページ構成は以下：

```
# YYYY Www（M/D〜M/D）

## 献立
### 平日
| 曜日 | メニュー | 所要時間 | 備考 |
### 休日

## 仕込み（日曜30分）
- [ ] ...

## 買い物リスト
### 肉
### 野菜
### 乾物・缶詰
### ※家にあるか確認

## 翌週繰越候補
- 使い切れない見込みの食材を明示

## メモ・振り返り
- 実食後の気づき・買い物のミス・申し送り
```

#### 買い物リスト部分のルール
- 食材をカテゴリ別（肉・魚 / 野菜 / 乾物・缶詰 / その他）に整理
- 既に持っていそうな調味料（醤油・塩・砂糖など基本のもの）は別セクション「※家にあるか確認」に分ける
- **半端食材の使い切り確認**：1レシピでしか使わない食材は「⚠️余る可能性」と明示
- **翌週繰越候補**：使い切れない見込みの食材は別セクションに明示し、翌週の献立組成時に優先消費する

## やらないこと

- レシピを勝手に削除しない（「削除して」と明示的に言われた場合のみ）
- メモ欄・訪問履歴の過去の気づきを上書きしない（追記のみ）
- 食材の好き嫌いを推測しない（明示されてないなら聞く）
- **生トマトを使ったレシピは出さない**（加熱トマトはOK）
- 自炊できなかった日を責める表現は使わない（「無理しない」「育てる方向でOK」のフレームを返す）
